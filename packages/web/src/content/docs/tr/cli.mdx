exos-agent CLI herhangi bir argüman olmadan çalıştırıldığında varsayılan olarak [TUI](/docs/tui)'yi başlatır.

```bash
exos-agent
```

Ancak bu sayfada belgelendiği gibi komutları da kabul eder. Bu, exos-agent ile programlı olarak etkileşim kurmanıza olanak tanır.

```bash
exos-agent run "Explain how closures work in JavaScript"
```

---

### tui

exos-agent terminal kullanıcı arayüzünü başlatır.

```bash
exos-agent [project]
```

#### Bayraklar

| Bayrak                                   | Kısa | Açıklama                                                                    |
| ---------------------------------------- | ---- | --------------------------------------------------------------------------- |
| <nobr><code>{"--continue"}</code></nobr> | `-c` | Son oturuma devam et                                                        |
| <nobr><code>{"--session"}</code></nobr>  | `-s` | Devam edecek oturum kimliği                                                 |
| <nobr><code>{"--fork"}</code></nobr>     |      | Devam ederken oturumu fork'lar (`--continue` veya `--session` ile kullanın) |
| <nobr><code>{"--prompt"}</code></nobr>   |      | Kullanılacak prompt                                                         |
| <nobr><code>{"--model"}</code></nobr>    | `-m` | provider/model biçiminde kullanılacak model                                 |
| <nobr><code>{"--agent"}</code></nobr>    |      | Kullanılacak agent                                                          |
| <nobr><code>{"--port"}</code></nobr>     |      | Dinlenecek port                                                             |
| <nobr><code>{"--hostname"}</code></nobr> |      | Dinlenecek host adı                                                         |

## Komutlar

exos-agent CLI ayrıca aşağıdaki komutlara da sahiptir.

---

### agent

exos-agent için agent'ları yönetin.

```bash
exos-agent agent [command]
```

---

### attach

`serve` veya `web` komutlarıyla başlatılan, halihazırda çalışan bir exos-agent arka uç sunucusuna bir terminal ekleyin.

```bash
exos-agent attach [url]
```

Bu, TUI öğesinin uzak bir exos-agent arka ucuyla kullanılmasına olanak tanır. Örneğin:

```bash
# Start the backend server for web/mobile access
exos-agent web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
exos-agent attach http://10.20.30.40:4096
```

#### Bayraklar

| Bayrak                                   | Kısa | Tanım                                                                                         |
| ---------------------------------------- | ---- | --------------------------------------------------------------------------------------------- |
| <nobr><code>{"--dir"}</code></nobr>      |      | TUI'yi başlatmak için çalışma dizini                                                          |
| <nobr><code>{"--continue"}</code></nobr> | `-c` | Son oturuma devam et                                                                          |
| <nobr><code>{"--session"}</code></nobr>  | `-s` | Devam edecek oturum açma bilgileri                                                            |
| <nobr><code>{"--fork"}</code></nobr>     |      | Devam ederken oturumu çatalla (`--continue` veya `--session` ile kullanın)                    |
| <nobr><code>{"--password"}</code></nobr> | `-p` | Temel kimlik doğrulama parolası (varsayılan: `EXOS_AGENT_SERVER_PASSWORD`)                      |
| <nobr><code>{"--username"}</code></nobr> | `-u` | Temel kimlik doğrulama kullanıcı adı (varsayılan: `EXOS_AGENT_SERVER_USERNAME` veya `exos-agent`) |

#### create

Özel yapılandırmaya sahip yeni bir agent oluşturun.

```bash
exos-agent agent create
```

Bu komut, özel sistem istemi ve araç yapılandırmasıyla yeni bir agent oluşturma konusunda size yol gösterecektir.

#### list

```bash
exos-agent agent list
```

### auth

Sağlayıcılar için kimlik bilgilerini ve oturum açmayı yönetme komutu.

```bash
exos-agent auth [command]
```

---

#### login

exos-agent, [Models.dev](https://models.dev) adresindeki sağlayıcı listesi tarafından desteklenmektedir, dolayısıyla kullanmak istediğiniz herhangi bir sağlayıcı için API anahtarlarını ayarlamak üzere `exos-agent auth login` kullanabilirsiniz. Bu `~/.local/share/exos-agent/auth.json`'de saklanıyor.

```bash
exos-agent auth login
```

exos-agent başlatıldığında sağlayıcıları kimlik bilgileri dosyasından yükler. Ayrıca ortam değişkenlerindeki veya projenizdeki `.env` dosyasındaki anahtarları da kullanır.

---

#### list

Kimlik bilgileri dosyasında depolanan tüm kimliği doğrulanmış sağlayıcıları listeler.

```bash
exos-agent auth list
```

Veya kısa versiyonu.

```bash
exos-agent auth ls
```

---

#### logout

Bir sağlayıcıyı kimlik bilgileri dosyasından temizleyerek oturumunuzu kapatır.

```bash
exos-agent auth logout
```

#### auth

Ayrıca OAuth özellikli sunucuları ve bunların kimlik doğrulama durumlarını da listeleyebilirsiniz.

### models

Yapılandırılmış sağlayıcıların tüm mevcut modellerini listeleyin.

```bash
exos-agent models [provider]
```

Bu komut, yapılandırdığınız sağlayıcılardaki modelleri `provider/model` formatında listeler.

Bu, [config dosyanızda](/docs/config/) kullanmanız gereken tam model adını bulmak için kullanışlıdır.

Modelleri bu sağlayıcıya göre filtrelemek için isteğe bağlı olarak bir sağlayıcı kimliğini iletebilirsiniz.

```bash
exos-agent models anthropic
```

#### Bayraklar

| Bayrak                                  | Tanım                                                                       |
| --------------------------------------- | --------------------------------------------------------------------------- |
| <nobr><code>{"--refresh"}</code></nobr> | Modeller.dev'den model önbelleğini yenileyin                                |
| <nobr><code>{"--verbose"}</code></nobr> | Daha ayrıntılı model çıktısı kullanın (maliyetler gibi meta veriler içerir) |

Önbelleğe alınan model listesini güncellemek için `--refresh` bayrağını kullanın. Bu, bir sağlayıcıya yeni modeller eklendiğinde ve bunları exos-agent'da görmek istediğinizde kullanışlıdır.

```bash
exos-agent models --refresh
```

### run

Doğrudan bir istem ileterek exos-agent'u etkileşimli olmayan modda çalıştırın.

```bash
exos-agent run [message..]
```

Bu, komut dosyası oluşturma, otomasyon veya TUI'un tamamını başlatmadan hızlı bir yanıt istediğinizde kullanışlıdır. Örneğin.

```bash "exos-agent run"
exos-agent run Explain the use of context in Go
```

Ayrıca, her çalıştırmada MCP sunucusunun soğuk önyükleme sürelerini önlemek için çalışan bir `exos-agent serve` örneğine de ekleyebilirsiniz:

```bash
# Start a headless server in one terminal
exos-agent serve

# In another terminal, run commands that attach to it
exos-agent run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

| ---------------------------------------- | ---- | --------------------------------------------------------------------------------------------- |
| <nobr><code>{"--command"}</code></nobr>  |      | Çalıştırılacak komut, args için mesajı kullanın                                               |
| <nobr><code>{"--continue"}</code></nobr> | `-c` | Son oturuma devam et                                                                          |
| <nobr><code>{"--session"}</code></nobr>  | `-s` | Devam edecek oturum kimliği                                                                   |
| <nobr><code>{"--fork"}</code></nobr>     |      | Devam ederken oturumu fork'lar (`--continue` veya `--session` ile kullanın)                   |
| <nobr><code>{"--share"}</code></nobr>    |      | Oturumu paylaşın                                                                              |
| <nobr><code>{"--model"}</code></nobr>    | `-m` | provider/model biçiminde kullanılacak model                                                   |
| <nobr><code>{"--agent"}</code></nobr>    |      | Kullanılacak temsilci                                                                         |
| <nobr><code>{"--file"}</code></nobr>     | `-f` | Mesaja eklenecek dosya(lar)                                                                   |
| <nobr><code>{"--format"}</code></nobr>   |      | Biçim: varsayılan (biçimlendirilmiş) veya json (ham JSON olayları)                            |
| <nobr><code>{"--title"}</code></nobr>    |      | Oturumun başlığı (değer sağlanmazsa kısaltılmış bilgi istemi kullanılır)                      |

| <nobr><code>{"--thinking"}</code></nobr> |      | Düşünme bloklarını göster                                                                     |
| <nobr><code>{"--port"}</code></nobr>     |      | Yerel sunucunun bağlantı noktası (varsayılan olarak rastgele bağlantı noktasıdır)             |

### serve

API erişimi için headless bir exos-agent sunucusu başlatır. Tam HTTP arayüzü için [server docs](/docs/server) sayfasına bakın.

```bash
exos-agent serve
```

Bu, TUI arayüzü olmadan exos-agent işlevselliğine API erişimi sağlayan bir HTTP sunucusunu başlatır. HTTP temel kimlik doğrulamasını etkinleştirmek için `EXOS_AGENT_SERVER_PASSWORD` öğesini ayarlayın (kullanıcı adı varsayılan olarak `exos-agent` şeklindedir).

#### Bayraklar

| Bayrak                                   | Tanım                                      |
| ---------------------------------------- | ------------------------------------------ |
| <nobr><code>{"--port"}</code></nobr>     | Dinlenecek bağlantı noktası                |
| <nobr><code>{"--hostname"}</code></nobr> | Dinlenecek ana bilgisayar adı              |
| <nobr><code>{"--mdns"}</code></nobr>     | mDNS bulmayı etkinleştir                   |
| <nobr><code>{"--cors"}</code></nobr>     | CORS'a izin verecek ek tarayıcı kaynakları |

```bash
exos-agent export [sessionID]
```

### import

Bir JSON dosyasından veya exos-agent paylaşımından URL oturum verilerini içe aktarın.

```bash
exos-agent import <file>
```

Yerel bir dosyadan veya exos-agent paylaşımından (URL) içe aktarabilirsiniz.

```bash
exos-agent import session.json
exos-agent import https://opncd.ai/s/abc123
```
