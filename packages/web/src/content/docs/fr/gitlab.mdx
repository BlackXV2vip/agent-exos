Exos Agent s'intègre à votre flux de travail GitLab via votre pipeline CI/CD GitLab ou avec GitLab Duo.

Dans les deux cas, Exos Agent fonctionnera sur vos runners GitLab.

---

## GitLab CI

Exos Agent fonctionne dans un pipeline GitLab régulier. Vous pouvez l'intégrer dans un pipeline en tant que [composant CI](https://docs.gitlab.com/ee/ci/components/)

Ici, nous utilisons un composant CI/CD créé par la communauté pour Exos Agent — [nagyv/gitlab-exos-agent](https://gitlab.com/nagyv/gitlab-exos-agent).

---

### Fonctionnalités

- **Utiliser une configuration personnalisée par tâche** : configurez Exos Agent avec un répertoire de configuration personnalisé, par exemple `./config/#custom-directory` pour activer ou désactiver la fonctionnalité pour chaque appel d'Exos Agent.
- **Configuration minimale** : le composant CI configure Exos Agent en arrière-plan, il vous suffit de créer la configuration Exos Agent et le prompt initial.
- **Flexible** : le composant CI prend en charge plusieurs entrées pour personnaliser son comportement

---

### Installation

1. Stockez votre authentification Exos Agent JSON en tant que variables d'environnement CI de type fichier sous **Paramètres** > **CI/CD** > **Variables**. Assurez-vous de les marquer comme « Masquées et protégées ».
2. Ajoutez ce qui suit à votre fichier `.gitlab-ci.yml`.

   ```yaml title=".gitlab-ci.yml"
   include:
     - component: $CI_SERVER_FQDN/nagyv/gitlab-exos-agent/exos-agent@2
       inputs:
         config_dir: ${CI_PROJECT_DIR}/exos-agent-config
         auth_json: $EXOS_AGENT_AUTH_JSON # The variable name for your ExosAgent authentication JSON
         command: optional-custom-command
         message: "Your prompt here"
   ```

Pour plus d'entrées et de cas d'utilisation [consultez la doc](https://gitlab.com/explore/catalog/nagyv/gitlab-exos-agent) pour ce composant.

## GitLab Duo

Exos Agent s'intègre à votre flux de travail GitLab.
Mentionnez `@exos-agent` dans un commentaire et Exos Agent exécutera les tâches au sein de votre pipeline CI GitLab.

---

### Fonctionnalités

- **Triage d'issues** : demandez à Exos Agent d'examiner une issue et de vous l'expliquer.
- **Correction et implémentation** : demandez à Exos Agent de résoudre une issue ou d'implémenter une fonctionnalité.
  Cela créera une nouvelle branche et déclenchera une merge request avec les modifications.
- **Sécurisé** : Exos Agent fonctionne sur vos runners GitLab.

---

### Installation

Exos Agent s'exécute dans votre pipeline CI/CD GitLab. Voici ce dont vous aurez besoin pour le configurer :

:::tip
Consultez la [**documentation GitLab**](https://docs.gitlab.com/user/duo_agent_platform/agent_assistant/) pour obtenir des instructions à jour.
:::

1.  Configurez votre environnement GitLab
2.  Configurer CI/CD
3.  Obtenez la clé API d'un fournisseur de modèles d'IA
4.  Créer un compte de service
5.  Configurer les variables CI/CD
6.  Créez un fichier de configuration de flux, voici un exemple :

        <details>

    <summary>Configuration du flux</summary>

    ```yaml
    image: node:22-slim
    commands:
      - echo "Installing exos-agent"
      - npm install --global exos-agent
      - echo "Installing glab"
      - export GITLAB_TOKEN=$GITLAB_TOKEN_EXOS_AGENT
      - apt-get update --quiet && apt-get install --yes curl wget gpg git && rm --recursive --force /var/lib/apt/lists/*
      - curl --silent --show-error --location "https://raw.githubusercontent.com/upciti/wakemeops/main/assets/install_repository" | bash
      - apt-get install --yes glab
      - echo "Configuring glab"
      - echo $GITLAB_HOST
      - echo "Creating ExosAgent auth configuration"
      - mkdir --parents ~/.local/share/exos-agent
      - |
        cat > ~/.local/share/exos-agent/auth.json << EOF
        {
          "anthropic": {
            "type": "api",
            "key": "$ANTHROPIC_API_KEY"
          }
        }
        EOF
      - echo "Configuring git"
      - git config --global user.email "exos-agent@gitlab.com"
      - git config --global user.name "ExosAgent"
      - echo "Testing glab"
      - glab issue list
      - echo "Running ExosAgent"
      - |
        exos-agent run "
        You are an AI assistant helping with GitLab operations.

        Context: $AI_FLOW_CONTEXT
        Task: $AI_FLOW_INPUT
        Event: $AI_FLOW_EVENT

        Please execute the requested task using the available GitLab tools.
        Be thorough in your analysis and provide clear explanations.

        <important>
        Please use the glab CLI to access data from GitLab. The glab CLI has already been authenticated. You can run the corresponding commands.

        If you are asked to summarize an MR or issue or asked to provide more information then please post back a note to the MR/Issue so that the user can see it.
        You don't need to commit or push up changes, those will be done automatically based on the file changes you make.
        </important>
        "
      - git checkout --branch $CI_WORKLOAD_REF origin/$CI_WORKLOAD_REF
      - echo "Checking for git changes and pushing if any exist"
      - |
        if ! git diff --quiet || ! git diff --cached --quiet || [ --not --zero "$(git ls-files --others --exclude-standard)" ]; then
          echo "Git changes detected, adding and pushing..."
          git add .
          if git diff --cached --quiet; then
            echo "No staged changes to commit"
          else
            echo "Committing changes to branch: $CI_WORKLOAD_REF"
            git commit --message "Codex changes"
            echo "Pushing changes up to $CI_WORKLOAD_REF"
            git push https://gitlab-ci-token:$GITLAB_TOKEN@$GITLAB_HOST/gl-demo-ultimate-dev-ai-epic-17570/test-java-project.git $CI_WORKLOAD_REF
            echo "Changes successfully pushed"
          fi
        else
          echo "No git changes detected, skipping push"
        fi
    variables:
      - ANTHROPIC_API_KEY
      - GITLAB_TOKEN_EXOS_AGENT
      - GITLAB_HOST
    ```

        </details>

Vous pouvez vous référer aux [GitLab CLI agents docs](https://docs.gitlab.com/user/duo_agent_platform/agent_assistant/) pour des instructions détaillées.

### Exemples

Voici quelques exemples de la façon dont vous pouvez utiliser Exos Agent dans GitLab.

:::tip
Vous pouvez configurer pour utiliser une phrase de déclenchement différente de `@exos-agent`.
:::

- **Expliquer une issue**

Ajoutez ce commentaire dans une issue GitLab.

```
  @exos-agent explain this issue
```

Exos Agent lira l'issue et répondra avec une explication claire.

- **Résoudre une issue**

Dans une issue GitLab, dites :

```
  @exos-agent fix this
```

Exos Agent créera une nouvelle branche, mettra en œuvre les modifications et ouvrira une merge request avec les modifications.

- **Revue de merge request**

Laissez le commentaire suivant sur une merge request GitLab.

```
  @exos-agent review this merge request
```

Exos Agent examinera la merge request et fournira des commentaires.
